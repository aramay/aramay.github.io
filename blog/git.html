What are the benefits of version control

Feb-12-2016

Developer can use it to achieve following goals

  • maintain different version of software
  • revert changes
  • easy backup
  • more than one developer can work on project

How does git help you keep track of changes

Git is a widely used “source code management system” for software development. It is the management of changes to documents, computer programs, large web sites, and other collections of information. Git is like an App that runs locally on user's machine.

Git

  1. Repository

    2. A repository is a special directory which has all the features of version control system. You can create a local repository by

    • create a directory
    • type command - git init
    • or clone an existing repository by git clone /path/to/directory

  2. Workflow

    3. Your local directory consist of three trees:

    • working directory - which has all the actual files
    • add files to your repository which have been changed - git add *
      • actual commit would save those files in local repository - git commit -m “message”

  3. Push Changes
    4. To send these changes to remote repository,
    type command git push origin <⁢branch-name>

GitHub

GitHub is web based repository hosting service. A web based model allows multiple users to work on same project. This way all the changes are synced and developers across the globe can keep up with progress. This can be accomplished as follows.
  1. Update and Merge
    2. Developer 1 can get the latest repository by making a call
    pull request. git pull
  2. After making changes to his files, developer 1 can push changes, so repository is synced.
    3. git push origin master
  3. developer 2 and developer 3 can sync their local repositories by doing
    git pull (it will merge latest changes)

Why use GitHub to store your code ?

Version Control is a widely used “source code management system” for software development. It is the management of changes to documents, computer programs, large web sites, and other collections of information. GitHub is a repository hosting service. It’s flagship feature is “forking”. Forking is copying a repository from one user’s account to another. This enables you to take a project that you don’t have write access to and modify it under your own account. If you make changes you’d like to share, you can send a notification called a “pull request” to the original owner. That user can then, with a click of a button, merge the changes found in your repo with the original repo.
